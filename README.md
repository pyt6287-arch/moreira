# KeepSpace - Supabase Bookmark Manager (Vercel Edition)

Un gestor de marcadores rápido, minimalista y serverless, diseñado para ejecutarse enteramente en el cliente y desplegarse en **Vercel** directamente desde tu repositorio de GitHub.

## Estructura del Proyecto

- `index.html`: Estructura y maquetación responsiva con tema oscuro.
- `style.css`: Estilos modernos (Outfit font, animaciones y diseño premium).
- `main.js`: Lógica cliente y conexión con Supabase.
- `package.json`: Configuración de scripts de construcción (Vite) y dependencias.

## Configuración de Supabase

1. Crea un nuevo proyecto en [Supabase](https://supabase.com).
2. Ejecuta el script SQL en el **SQL Editor** de Supabase para crear la tabla de marcadores:
   ```sql
   create table bookmarks (
     id bigint generated by default as identity primary key,
     title text not null,
     url text not null,
     created_at timestamp with time zone default timezone('utc'::text, now()) not null
   );

   alter table bookmarks enable row level security;

   create policy "Permitir acceso público" on bookmarks for all to anon using (true) with check (true);
   ```

## Despliegue en Vercel

1. Sube este proyecto a tu repositorio de GitHub.
2. Inicia sesión en [Vercel](https://vercel.com) e importa el proyecto.
3. Vercel detectará automáticamente que es un proyecto de **Vite**.
4. En la sección **Environment Variables**, añade:
   - `VITE_SUPABASE_URL`: Tu URL del proyecto de Supabase.
   - `VITE_SUPABASE_KEY`: Tu clave anónima (`anon` public key).
5. Haz clic en **Deploy**.

## Ejecución Local

1. Copia `.env.example` a `.env` y rellena las credenciales de Supabase:
   ```bash
   cp .env.example .env
   ```
2. Instala dependencias y arranca el servidor de desarrollo:
   ```bash
   npm install
   npm run dev
   ```
3. Abre [http://localhost:5173](http://localhost:5173) en tu navegador.
